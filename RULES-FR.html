Règlement concours performance web

But du concours

Optimiser une page web existante et essayer d'obtenir la meilleur performance pour ces 3 catégories :

  • Meilleur score YOTTAA calculé avec le site web http://www.yottaa.com
  • Meilleur temps onload et start render calculés à l'aide du site web http://www.webpagetest.org
  • Plus petit nombre de requêtes et poids du site, à l'aide du site web http://www.webpagetest.org

Pour les calculs réalisés avec http://www.webpagetest.org, les paramètres seront : Paris, IE7, DSL.

Vos pages web optimisées seront à renvoyer sur notre FTP, ainsi tous les tests de performance seront réalisés sur le même serveur.

Page objet du concours

La société FNAC (FNAC DIRECT SA) a autorisé l'organisation du concours à prendre comme page d'exemple l'une des pages de leur site web http://www.fnac.com. Il s'agit d'un des plus grands sites de e-commerce Français.

Nous remercions donc la FNAC pour sa confiance dans notre projet de concours performance web.

L'url originale de la page choisie est : http://www.fnac.com/enfants.asp.

Vous trouverez sur la page inscription une version packagée de cette page web pour le concours.

Fonctionnalités à conserver

En plus du fait que le résultat graphique doit être strictement identique à ce qui existe actuellement, vous veillerez à ce que vos optimisations conservent les fonctionnalités de la page web originale.

En effet, la page web http://www.fnac.com/enfants.asp comporte des fonctionnalités que vous devrez conserver, soit :

  • Le champs recherche et l'autocomplétion (essayez de rentrer "son" et il vous propose des pages correspondantes)
  • Le défilement doux lorsqu'on clique sur la flèche à droite de "mon panier"
  • Slider central avec changement automatique et manuel des images
  • Les menus du haut et sous menus au survol
  • Les indications "x produits trouvés" qui sont récupérées en ajax
  • Le système de tracking doit fonctionner (s_code)
  • Toutes autres fonctionnalités de la page originale que vous pourriez identifier, nous serons aussi là pour répondre à vos questions

Moyens fournis

Lorsque votre inscription sera validée, vous obtiendrez sous 24h vos identifiants FTP pour envoyer vos fichiers sur nos serveurs.

Chaque participant aura accès à son répertoire FTP de base sur 4 domaines différents :

  • entries.webperf-contest.com
  • s1.webperf-contest.com
  • s2.webperf-contest.com
  • s3.webperf-contest.com

Ceci vous permet par exemple de faire des appels d'images sur le domaine s1.webperf-contest.com plutôt que sur entries.webperf-contest.com.

Les tests de performance pour les classements seront réalisés sur votre fichier "index.html" sur le domaine entries.webperf-contest.com (domaine principal). Cela signifie que vous ne devez pas changer le nom du fichier html principal (index.html).

Triche

  • Pas d'abus ! Afficher un JPG à la place du site, c'est tricher (si si, même si vous enlevez les données EXIF)
  • Pas de scripting côté serveur
  • L'expérience utilisateur doit être préservée (et vous pouvez l'améliorer), par exemple :
    • Une fois la page chargée, la page doit être aussi fonctionnelle qu'à l'origine
    • L'accessibilité ne doit pas être sacrifiée (vous pouvez même l'améliorer)
    • Les optimisations pour référencement doivent être préservées (vous pouvez également les améliorer)
  • Utilisez nos serveurs, vous ne pouvez héberger vos fichiers ailleurs que sur *.webperf-contest.com
  • Vous avez le droit d'utiliser un fichier .htaccess pour modifier la configuration de votre répertoire
  • CSS1-2-3, Canvas, SVG, JavaScript (inline, externe, minifié ...), dataURIs, MHTML, defer, async, *insérez n'importe quelle technique de dingue ici* ... tout cela est AUTORISÉ et même encouragé

La règle principale est : agissez comme s'il s'agissait de votre site, de votre client et de votre design. Tout sacrifier pour la performance web ne sert à rien.

Nous croyons aux technique d'enrichissement progressif et de dégradation gracieuse, vous pouvez donc livrer une page qui n'est pas conforme au pixel près de l'original, tant que vous ne dégradez pas l'expérience utilisateur.

En cas de doute, demandez nous ! Nous regardons régulièrement les Gist de vos modifications, donc nous pourrons aussi vous contacter si nous pensons qu'il y a un problème.

Les juges auront le dernier mot pour déterminer ce qui est acceptable de ce qui ne l'est pas lors du départage final des candidats ayant des temps et des scores très proches.
